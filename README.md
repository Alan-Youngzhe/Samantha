# 别装 — AI 消费情报网络

> 一个人跟 Nick 聊天，一座城的消费真相就浮出来了。

---

## 这是什么

「别装」是一个 **AI-native 城市消费情报系统**的产品原型。

表面上，用户在和一个叫 **Nick** 的 AI 人格聊天——他用第三人称视角揭示你每一笔消费背后的真实动机。但在对话过程中，系统零摩擦地采集了精确到 **店铺→单品→价格→评价→动机** 的匿名消费情报，汇入一个城市级的消费知识网络。

**和大众点评的本质区别：** 大众点评的 AI（点仔）是在已有评价上做摘要——一个更聪明的搜索引擎。别装的数据不是用户主动写的评价，而是 **从自然对话中自动提取的、带消费动机的、经 Agent 回访验证的实时消费情报**。

```
传统平台：用户主动写评价 → AI 总结 → 所有人看到一样的结果
别装：    用户跟 Nick 聊天 → AI 自动提取情报 → 带动机、带可信度、个性化输出
```

## 产品弧线

```
个人层 ────────────────────────────────────────────────
第 1 天   Nick 观察你的消费，用镜头语言揭示真实动机
第 7 天   Nick 记住了你的每一个消费承诺，开始"打脸"
第 14 天  Nick 发现你的消费模式和触发链
第 30 天  你的消费镜像出现——用你自己的口吻说出你不愿承认的事

城市层 ────────────────────────────────────────────────
每一次对话  Nick 自动提取匿名单品评价，计算可信度，汇入情报网
有人问起时  "龙华天街的喜茶，5个人买过多肉葡萄，4个说不错——
            不过有1个是深夜冲动买的，第二天说太甜了"
每 5-10 轮  Nick 主动回访："上次那个海底捞，你后来又去了没？"
            → 验证评价真实性，更新可信度分数
```

## 核心能力

### 1. 消费动机引擎

每笔消费自动标记六种动机之一，不只是"好不好"，而是"为什么买"：

| 动机 | 说明 | 示例 |
|------|------|------|
| **need** | 真实需要 | 食堂盖饭 15 元 |
| **emotional** | 情绪消费 | 被骂后买奶茶 23 元 |
| **impulse** | 冲动消费 | 小红书种草台灯 198 元 |
| **social** | 社交消费 | 同事都有的 AJ 1299 元 |
| **habitual** | 习惯消费 | 每天下午瑞幸 16 元 |
| **reward** | 自我奖励 | 加班后酸菜鱼外卖 48 元 |

### 2. 匿名消费情报采集

用户跟 Nick 聊天时，系统自动提取结构化评价（店铺 × 单品 × 价格 × 情感 × 动机 × 时段 × 地理位置），无需用户主动填写任何表单。

**三种采集机制：**
- **自然对话提取** — "喝了杯喜茶多肉葡萄，还行" → 自动结构化
- **好奇心追问** — "哪家的？" "点了什么？" "值吗？"（只问一个，不审问）
- **图片识别** — 小票/外卖截图 → Claude Vision 提取店铺、商品、价格

### 3. Trust Score（可信度评分）

每条匿名评价带一个 0-100 的 AI 可信度分数，基于：

| 维度 | 权重 | 原理 |
|------|------|------|
| 画像深度 | 40% | 用户聊了多久、有多少消费记录、多少模式/触发链 |
| 行为一致性 | 20% | 评价是否有对应的 [SPENDING] 消费记录 |
| 位置验证 | 20% | 用户 GPS 是否在店铺附近 |
| 基础信任 | 20% | 账龄、历史对话量 |

**为什么能抗刷：** 大众点评的评价是孤立的，机器人注册就能写。Nick 的每条评价背后有完整的用户画像——聊了多久、有什么消费模式、有几条触发链。空壳用户的评价天然低可信度。

### 4. 主动回访验证（Agent 行为）

Nick 不只是等用户输入。他会主动回访之前的消费：

```
[第1天]  用户: "海底捞番茄锅不错"  → trustScore: 65
[第5天]  Nick: "上次那个海底捞，你后来又去了没？"
         用户: "没有，太远了"
         → [VERIFY] sentiment: positive → neutral, verified: true, trust +10
```

验证过的评价比未验证的权重更高。这是真正的 **Agent 行为**——不只是响应，而是主动维护数据质量。

### 5. 个性化消费参考

Nick 给不同用户的参考信息是不同的：

```
对冲动型消费者：
  "你是今天第三个想在这买东西的人了。前两个是冲动消费。"

对习惯型消费者：
  "这家瑞幸上周有 5 个人买了生椰拿铁，3 个是习惯性消费。你也是。"

对情绪消费者：
  "多肉葡萄这东西，心情好的时候喝的人都说不错，焦虑的时候买的基本都后悔。你现在什么状态？"
```

### 6. 五层记忆系统

```
L1  原始记录 — 每次对话 + 时间戳 + 图片描述
L2  消费画像 — 模式识别（冲动/情绪/习惯/社交/理性）
L3  承诺追踪 — "不再买奶茶" → 3天后打脸
L4  消费人格 — 触发链、借口、恐惧、矛盾
L5  消费流水 — 金额 × 类别 × 动机 × 地理位置
```

### 7. Mirror 镜像系统

极低频触发（需要足够的画像积累），用用户自己的口吻说出消费真相：

> "其实我知道那杯奶茶不是因为渴。是因为刚被甲方骂了。我又说'犒劳自己'了。我自己都听腻了这个借口。"

## AI 标签体系

Nick 在对话中静默输出结构化标签，前端自动解析处理：

| 标签 | 格式 | 用途 |
|------|------|------|
| `[SPENDING]` | 金额\|类别\|动机 | 消费记录入库 |
| `[REVIEW]` | 店铺\|单品\|价格\|sentiment\|评价 | 匿名情报采集 |
| `[VERIFY]` | 店铺\|单品\|new_sentiment | 回访验证更新 |
| `[COMMITMENT]` | 承诺内容 | 承诺追踪 |
| `[PATTERN]` | 类别\|描述 | 消费模式识别 |
| `[USER_TRAIT]` | 类别\|描述 | 消费人格标签 |
| `[TRIGGER_CHAIN]` | 触发场景\|消费行为 | 触发→消费链 |

## Demo 场景

### 场景 1：消费审视 + 情报采集

```
用户：中午在喜茶买了杯多肉葡萄，22块

Nick：又是她。龙华天街的喜茶。22块的多肉葡萄——说是路过顺便买，
但她的手机定位显示她绕了两条街。不是顺路，是惦记。
不过这家的多肉葡萄，上周有3个人买过，2个说不错。你运气还行。

[SPENDING]: 22|coffee|habitual
[REVIEW]: 喜茶|多肉葡萄|22|positive|路过忍不住买的
```

### 场景 2：消费意图拦截 + 附近情报

```
用户：想去海底捞

Nick：龙华天街那家？上次有人花了175，排了一个半小时，跟我说不值。
不过番茄锅评价还行。你确定是想吃，不是不想做饭？
```

### 场景 3：主动回访验证

```
Nick：说起来，上周你说海底捞不错。后来又去了没？

用户：没有，太远了其实

Nick：行。"不错"减去"太远了"等于"其实也没那么想去"。
→ [VERIFY]: 海底捞|番茄锅底|neutral
```

### 场景 4：Mirror 出现

```
[她的内心]：其实我知道那杯奶茶不是因为渴。是因为刚被甲方骂了。
我又说"犒劳自己"了。我自己都听腻了这个借口。
```

## 与大众点评的对比

| 维度 | 大众点评（点仔 AI） | 别装（Nick） |
|------|---------------------|--------------|
| 数据来源 | 用户主动写评价 | 对话自动提取（零摩擦） |
| 本质 | 已有评价的搜索+摘要 | **数据采集器** — 每次聊天产生新数据 |
| 动机数据 | ❌ 不知道为什么买 | ✅ 情绪/冲动/习惯/社交/需要/奖励 |
| 可信度 | 靠举报/算法猜 | **画像深度 × 行为一致性 × 位置验证 × 回访确认** |
| 数据维护 | 写完就不管了 | **Agent 主动回访验证** |
| 个性化 | 所有人看到一样的 | 根据你的消费性格定制参考 |
| 反刷单 | 难（评价是孤立的） | 天然抗刷（每条评价背后有完整画像） |

## 启动项目

```bash
npm install

# 创建 .env.local
ANTHROPIC_AUTH_TOKEN=your-api-key
ANTHROPIC_BASE_URL=https://your-api-proxy   # 可选

npm run dev
```

打开 http://localhost:3000

- **主界面** — 和 Nick 聊天，消费情报自动采集
- **/report** — Nick 的笔记，消费真相报告
- **/heatmap** — 消费热力图（B 端 demo，不在用户侧边栏）
