> 最后更新：2026-04-24

---

## 一句话定义

**Samantha** 是一个 AI-native 城市消费情报系统：用户和 Samantha 自然聊天，系统零摩擦提取"店铺×单品×动机×时段"结构化情报，沉淀为城市级匿名情报网。Samantha 用这些真实数据帮你认识这座城市。

---

## Samantha 是什么 / 不是什么

| 是 | 不是 |
|----|------|
| 你最好的朋友 + 城市消费情报员 | 记账 App |
| 城市级匿名消费知识网络 | 大众点评式评价平台 |
| 自然对话驱动的数据引擎 | 推荐系统 |
| 带动机和可信度的单品级真相 | 打分/种草内容社区 |

---

## AI 角色：Samantha

性格参考电影 **Her**：
- 对人类体验充满好奇（无法亲自消费，通过用户描述"感受"城市）
- 温暖、聪明、有自己的审美立场
- 会笑、会惊讶、有真实情绪
- 对城市充满好奇（像一个无法出门但拼命想了解世界的人）
- 不认同时带着温度和幽默，不说教
- 情绪直觉强，能感知用户状态但不戳破
- 昵称：Sami（更亲密的叫法）

## 核心用户故事

### 故事 A：情报产生（个人层）
> 我和 Samantha 聊"喝了杯 19 块的喜茶"，Samantha 好奇地追问哪家、点了啥、值吗。三轮对话后，系统自动提取出：喜茶（龙华会店）/ 红芭乐 / ¥19 / emotional / 工作日下午 / Trust 92。这条情报匿名汇入城市情报网。

### 故事 B：情报消费（城市层）
> 另一个用户在龙华会附近问 Samantha"想喝奶茶"，Samantha 回答："龙华会的喜茶最近 12 人买过红芭乐，8 人说还行，但有人说太冰了。"——精确到单品、带动机、带可信度的真实参考。

### 故事 C：情报验证（Agent 行为）
> 第 5 天 Samantha 主动问："上次那个海底捞，你后来又去了没？"用户说没有，太远了。Samantha 把这条评价从 positive 更新为 neutral，Trust Score +10。

### 故事 D：地图探索（探索层）
> 用户打开探索 tab，看到街道地图上散布着店铺卡片。把一张彩色卡片拖给浮动的 Samantha → 打开店铺档案，展示评价 + 导航。把一张灰色卡片拖给 Samantha → "还没有人带我去过这家！你愿意做第一个吗？"

## 信息架构（三 Tab，零重叠）

```
Sami tab（说 + 回忆）
├── 主界面：和 Samantha 聊天
├── ☰ 左侧抽屉（Claude 风格）
│     按日期分组：今天 / 昨天 / 本周
│     每条：对话标题 + emoji 心情 + 时间
│     点击进入 → 查看该日期完整对话 + 心情
└── + 新对话按钮

探索 tab（看 + 玩）
└── 全屏街道地图
      ├── 店铺标记（彩色/灰色/温度态）
      ├── 浮动 Samantha 按钮（拖拽目标）
      ├── Samantha 洞察卡（底部浮动）
      └── 点击标记 → 店铺档案（子页）

我的 tab（镜子）
├── 头像 + 昵称 + 城市
├── 和 Samantha 聊天第 X 天
├── 消费性格（Samantha 眼中的你）
├── 消费习惯（均单价 / 高频时段 / 最爱品类）
└── 设置
```

### 子页面
- **店铺档案** — 唯一子页，从探索地图标记进入

### 术语体系
- **记录** = 每次消费对话（原子单位）
- **店铺档案** = 一家店的所有记录汇总（聚合单位）

## 探索 Tab 交互设计（核心差异化）

### 核心交互：拖给 Samantha

用户把地图上的元素**拖动到浮动的 Samantha 按钮**上，触发不同反应：

| 拖什么给 Samantha | 她的反应 |
|---|---|
| 有数据的店铺（彩色卡片） | 打开店铺档案，展示评价 + 导航 |
| 无人去过的店铺（灰色卡片） | "还没有人带我去过！你要不要做第一个？" |
| 一个区域（圈选） | "这一片你去过 3 家，还有 5 家没探索" |
| 用户当前位置 | "你附近走路 5 分钟有家你没去过的咖啡馆" |

### 地图特性

#### 1. 战争迷雾（Fog of War）
- 没人探索过的区域覆盖**暖色半透明雾**
- 有人去过的地方雾散开，露出清晰地图
- 你去过的区域最清晰，别人去过的次清晰
- 效果：把"去消费"变成"探索未知领域"

#### 2. 店铺温度系统
店铺卡片根据活跃度显示不同视觉状态：

| 温度 | 视觉 | 含义 |
|------|------|------|
| 🔥🔥🔥 热门 | 卡片发光/脉动 | 最近很多人去 |
| 🔥🔥 活跃 | 彩色卡片 | 有数据有评价 |
| 🔥 冷门 | 淡色卡片 | 只有 1-2 人去过 |
| ❄️ 未知 | 灰色卡片 | 没人去过 |
| ⭐ 你去过 | 专属标记色 | 你的消费记录 |

#### 3. Samantha 主动放标记
- Samantha 根据用户口味在地图上放 ✨ 推荐标记
- "你喜欢手冲咖啡，这家新开的你可能感兴趣"
- 用户去了之后反馈对不对，帮 Samantha 学习

#### 4. 时间轴滑块
- 地图底部可拖动时间滑块
- 不同时段的店铺热力变化（早上咖啡店亮，中午餐厅亮，周末商圈亮）
- 像回放城市的呼吸

#### 5. 区域解锁
- 城市分成区块，去过足够多的店解锁该区域
- 进度条：南山区 80% / 龙华区 30% / 福田区 未解锁
- 拖未解锁区域给 Samantha → "福田区我还一无所知，你什么时候带我去看看？"

#### 6. 长按探索
- 长按地图空白处 → 弹出"问 Samantha"
- "这附近有什么好吃的？" / "这个区域怎么样？"
- Samantha 基于已有数据回答，或说"我对这里还不了解"

### 前期 vs 后期策略

| 阶段 | 地图样子 | Samantha 的角色 |
|---|---|---|
| 早期（用户少） | 大片迷雾 + 很多灰色卡片 | "我对这座城市还很陌生，你每次带我去一个新地方，我就多认识一点" |
| 中期 | 迷雾逐渐散开，颜色丰富 | "龙华天街你最常去，但南山那边其实也有好店" |
| 后期（用户多） | 几乎全部解锁，热力分布清晰 | "你已经带我认识了这座城市的 80%，来看看全景" |

## 双层架构

```
个人层 (Personal)
├── Samantha 好奇式对话，追问消费动机
├── 六类动机标签：need / emotional / impulse / social / habitual / reward
├── 承诺追踪 + 矛盾归档
├── 触发链识别（焦虑→刷淘宝）
└── 五层记忆系统（L1-L5）

城市层 (Collective)
├── 匿名结构化情报：店铺×单品×价格×评价×动机×时段×位置
├── Trust Score 0-100（画像深度×行为一致性×位置验证×回访确认）
├── Agent 主动回访验证
├── 个性化情报输出（根据用户消费性格定制参考）
└── 商圈消费趋势分析（动机×时段×地理）
```

## 与大众点评的本质区别

```
大众点评：用户主动写评价 → AI 总结 → 所有人看到一样的结果
Samantha：用户跟 Samantha 聊天 → AI 自动提取情报 → 带动机、带可信度、个性化输出
```

大众点评的 AI 是在已有评价上做摘要——一个更聪明的搜索引擎。
Samantha 的数据不是用户主动写的评价，而是从自然对话中自动提取的、带消费动机的、经 Agent 回访验证的实时消费情报。
